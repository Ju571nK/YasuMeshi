## 1. 프로젝트 개요

### 앱 한 줄 정의

"지금 있는 곳에서, 가장 싼 밥집과 카페를 바로 찾아주는 앱"

### 배경 및 문제 정의

일본의 기존 식당 검색 앱(食べログ, ぐるなび 등)은 리뷰·분위기·예약 중심으로 설계되어 있어, **"지금 배고픈데 근처에서 제일 싼 곳"** 이라는 즉각적인 니즈에는 대응이 느리다. 구글맵은 위치 기반 검색은 강하지만 가격 순 정렬이 불가능하다.

やすめし는 이 공백을 노린다. **가격 정보가 확실한 가게만, 저렴한 순으로, 바로 보여주는** 심플한 앱.

## 2. 타깃 유저

- 점심시간에 빠르게 저렴한 식당을 찾는 직장인
- 절약을 의식하는 학생·젊은 층
- 관광객·외국인 거주자 (일본어 메뉴 가격 파악이 어려운 층)
- 커피 한 잔 값도 아끼고 싶은 모든 사람

---

## 3. 핵심 컨셉 & 유저 시나리오

### 컨셉 원칙

- **가격 없는 가게는 최하위** — 가격이 확실한 가게를 우선 노출
- **앱 열자마자 리스트** — 복잡한 설정 없이 즉시 결과 표시
- **탭 하나로 카테고리 전환** — 식사 ↔ 음료 ↔ 테이크아웃

### 시나리오 A — 배고프다, 싼 식당 빨리 찾기

```
앱 오픈
  → 자동 위치 감지
  → 🍜 식사 탭 (기본)
  → 도보 10분 이내 / 가격 낮은 순 리스트
  → 가게 카드 탭 → 대표 메뉴 가격 확인
  → [지도로 이동] → 구글맵 실행
```

### 시나리오 B — 커피 마시고 싶다, 가장 싼 곳 찾기

```
앱 오픈
  → ☕ 음료 탭 선택
  → 도보 5분 / 10분 / 15분 선택
  → 커피 가격 낮은 순 리스트
  → 가게 탭 → 「아메리카노 ¥250~」 확인
  → [지도로 이동] → 구글맵 실행
```

## 4. 주요 기능

### 4-1. 홈 화면

- 카테고리 탭: 🍜 식사 / ☕ 음료 / 🍱 테이크아웃
- 도보 시간 필터: 5분 / 10분 / 15분 (기본: 10분)
- 정렬: 가격 낮은 순 (기본) / 거리 순
- 리스트 뷰 / 지도 뷰 전환

### 4-2. 가게 카드

- 가게명 + 카테고리
- 대표 메뉴 가격 (¥XXX~)
- 도보 예상 시간
- [지도로 이동] 버튼 → 구글맵 딥링크

### 4-3. 가격 정렬 방침

- `priceRange` 있는 가게 → 상위 (가격 낮은 순)
- `priceRange` 없는 가게 → 최하위 ("가격 정보 없음" 표시)

## 5. 데이터 소스 전략

**MVP: Google Places API (New) 단독**

- `priceRange` 필드에서 실제 가격 범위 취득
- 위치 기반 검색, 카테고리 필터, 지도 연동 모두 Google 생태계 안에서 처리
- ホットペッパー 등 외부 API는 MVP에서 제외, v2 이후 커버리지 보완용으로 검토

**데이터 플라이휠 전략**

앱 사용자가 늘수록 가격 없는 가게가 최하위 노출 → 가게 주인 / 방문자가 Google Maps에 가격 자발 입력 → `priceRange` 커버리지 증가 → 앱 품질 향상의 선순환 구조. 앱이 직접 데이터를 수집하지 않아도 Google Maps 에코시스템을 간접 활성화.

## 6. 수익 모델

### 수익 구조 방침

```
MAU 0 ~ 5,000   → AdMob 배너 광고로 API 비용 충당
MAU 5,000 이전  → 데이터 판매 프레임 준비 완료 목표
MAU 5,000+      → 데이터 판매 수익 본격화
```

### 1차 수익 — AdMob 배너 광고 (API 비용 충당 목적)

- Google AdMob 탑재
- 일본 eCPM 아시아 최상위권 → 광고 단가 유리
- 점심·저녁 시간대 CTR 집중
- MAU 1,000명 기준: API 비용 ~$10, 광고 수익 월 ¥3,000~6,000 → 충당 가능

### 2차 수익 — 행동 데이터 판매 (MAU 5,000 이전 프레임 완성 목표)

익명화·집계화된 데이터를 외부에 제공:

| 데이터 종류 | 잠재 구매처 |
| --- | --- |
| 지역별 / 시간대별 가격대 수요 | 부동산사, 상업시설 개발사 |
| 음식 카테고리 인기도 | 식품 제조사, 프랜차이즈 본사 |
| 역세권별 가성비 검색 패턴 | 음식점 출점 컨설팅 |
| 도보 필터 선택 패턴 | 마케팅사, 광고대행사 |

> 개인정보보호법 준수를 위해 데이터는 반드시 익명화·집계 처리 후 제공. プライバシーポリシーに明示 필수.

## 8. 경쟁 우위 및 차별점

- **가격 순 정렬** — 구글맵·食べログ에는 없는 기능
- **즉각성** — 앱 열자마자 결과. 검색·필터 없이
- **데이터 플라이휠** — 사용자 증가가 데이터 품질 향상으로 직결되는 구조
- **심플한 수익 구조** — AdMob + 데이터 판매로 개발·운영 비용 회수 가능
