## 핵심 방침

MAU 5,000 도달 **이전**에 데이터 판매 인프라와 영업 프레임을 완성한다.

앱이 커진 다음에 준비하면 늦다 — 데이터 설계는 **출시 첫날부터** 시작해야 한다.

---

## Phase 1 — 출시 즉시: 데이터 수집 설계 (MAU 0~)

수익화 전이라도 반드시 수집해야 할 이벤트:

| Firebase 이벤트명 | 수집 내용 | 판매 가치 |
| --- | --- | --- |
| `category_tap` | 식사/음료/테이크아웃 탭 + 시간대 | 시간대별 식욕 수요 |
| `walk_filter_select` | 5분/10분/15분 선택 + 위치(역명) | 역세권별 행동반경 수요 |
| `shop_card_tap` | 클릭된 가게 가격대 + 카테고리 | 지역별 가성비 수요 |
| `map_navigate` | 지도 이동 실행 여부 | 실구매 전환율 |
| `session_time` | 앱 사용 시간대 | 식사 패턴 분석 |

> 모든 이벤트는 **개인 식별 불가** 형태로 수집. 위치는 역명/구 단위로만 기록.
>

## Phase 2 — MAU 500 도달: 데이터 집계 파이프라인 구축

```
Firebase Analytics
  → BigQuery Export (자동 연동)
    → 집계 쿼리 작성
      → 월간 리포트 템플릿 생성
        → 샘플 데이터셋 준비 (영업용)
```

**만들어야 할 리포트 형태:**

- 도쿄 23구별 × 시간대별 저가 식당 수요 히트맵
- 카테고리별 (라멘/카레/카페 등) 인기 역세권 TOP 10
- 도보 5분 vs 10분 vs 15분 선택 비율 추이

## Phase 3 — MAU 1,000 도달: 잠재 구매처 영업 시작

### 타깃 구매처 및 접근 방법

| 구매처 유형 | 필요한 데이터 | 접근 채널 |
| --- | --- | --- |
| 음식점 출점 컨설팅 회사 | 역세권별 가성비 수요, 카테고리 공백 | LinkedIn, 직접 메일 |
| 부동산 개발사 (상업시설) | 지역별 점심 수요 밀도 | 부동산 전시회, 소개 |
| 프랜차이즈 본사 | 카테고리별 인기 지역 | 가맹 담당자 직접 접촉 |
| 식품 마케팅사 | 시간대별 음식 카테고리 수요 | 대행사 파트너십 |

### 데이터 판매 형태

- **월정액 리포트 구독**: 월 ¥30,000~100,000
- **커스텀 분석 의뢰**: 건당 ¥50,000~200,000
- **API 형태 제공** (MAU 충분 시): 별도 협의
