## 결정 사항

- **ホットペッパー API는 MVP 단계에서 사용하지 않는다**
- **가격 데이터가 없는 가게는 리스트에 표시하지 않는다**
- **Google Places API 단독으로 MVP를 시작한다**

---

## 판단 근거

### Google Places API (New) — priceRange 필드

```json
{
  "priceRange": {
    "startPrice": { "units": 500 },
    "endPrice": { "units": 1000 }
  }
}
```

- 이 필드가 존재하는 가게만 리스트에 표시
- 필드가 없으면 → 조용히 제외 ("가격 정보 없음" 표시도 하지 않음)
- priceLevel만 있고 priceRange가 없는 가게도 제외

### ホットペッパー API 포지셔닝

| 항목 | 내용 |
| --- | --- |
| MVP 단계 | 사용 안 함 |
| 향후 v2 검토 | 커버리지 부족 시 보완용으로 추가 고려 |
| 이유 | 무료지만 등록 가게 한정, 메뉴 단위 데이터 품질 불균일 |

---

## 필터링 로직 방침

```
가게 데이터 수신
  └── priceRange 존재? 
        ├── YES → 리스트에 표시 (가격 낮은 순 정렬)
        └── NO  → 표시 안 함 (숨김 처리)
```

> 사용자 입장에서는 "가격을 모르는 가게"가 리스트에 섞이는 것보다, 적더라도 가격이 확실한 가게만 보이는 게 낫다.
>
