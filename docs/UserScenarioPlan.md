## 결정 사항

- 시나리오 1 (식사) + 시나리오 2 (음료)는 **하나의 앱에 통합** — 카테고리 필터로 전환
- 반경 설정은 **도보 시간 기준** (5분 / 10분 / 15분)

---

## 시나리오 1 — 배고프다, 싼 식당 빨리 찾기

**컨셉:** 배가 고픈 상태에서 빠르게 주변 저렴한 식당을 찾고 바로 이동

**유저 플로우:**

1. 앱 오픈 → 자동 위치 감지
2. 홈 화면에 카테고리 필터 표시 (🍜 식사 / ☕ 음료 / 🍱 테이크아웃 등)
3. 기본값: 도보 10분 이내, 가격 낮은 순 리스트
4. 가게 카드 탭 → 메뉴 + 가격 미리보기
5. 「지도로 보기」버튼 → 구글맵 / Apple Maps 앱으로 이동

**핵심 UX 포인트:**

- 앱 열자마자 리스트가 바로 보여야 함 (로딩 최소화)
- 가게 카드에 대표 메뉴 가격 1~2개 표시
- 도보 시간 탭으로 빠르게 전환 (5분 | 10분 | 15분)

**데이터 소스:** Google Places API + ホットペッパーグルメ API

---

## 시나리오 2 — 커피 마시고 싶다, 가장 싼 곳 찾기

**컨셉:** 음료 카테고리 선택 후 최저가 순으로 빠르게 위치 확인

**유저 플로우:**

1. 앱 오픈 → 카테고리에서 ☕ 음료 선택
2. 도보 시간 필터 선택 (기본: 10분)
3. 커피 / 음료 가격 낮은 순 리스트 표시
4. 가게 탭 → 대표 음료 가격 확인
5. 지도 앱으로 이동

**핵심 UX 포인트:**

- 카테고리 전환이 탭 한 번으로 가능해야 함
- 편의점 커피 (セブン, ローソン 등) 도 포함 검토 필요
- 가격 표시: 「아메리카노 ¥250~」 형식

**데이터 소스:** Google Places API (카페 카테고리) + ホットペッパー

---

## 통합 UX 구조

```
홈 화면
├── 카테고리 탭: 🍜 식사 | ☕ 음료 | 🍱 테이크아웃
├── 도보 시간 필터: 5분 | 10분 | 15분
├── 정렬: 가격 낮은 순 (기본) | 거리 순
└── 리스트 뷰 / 지도 뷰 전환 버튼

가게 카드
├── 가게명 + 카테고리
├── 대표 메뉴 가격 (¥XXX~)
├── 도보 시간 표시
└── [지도로 이동] 버튼
```
